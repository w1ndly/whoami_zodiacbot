# CHANGELOG



## v1.0.0 — Коммерческая версия

### Добавлено


- Добавлена финальная обработка Robokassa ResultURL.
- После успешного callback от Robokassa заказ получает статус paid.
- Оплаченные проверки автоматически начисляются пользователю.

* Добавлена витрина пополнения проверок.
* Добавлены три пакета проверок:
  * 10 проверок — 100 Stars;
  * 25 проверок — 220 Stars;
  * 50 проверок — 390 Stars.
* Подключена оплата через Telegram Stars.
* После успешной оплаты бонусные проверки автоматически начисляются пользователю.
* Добавлен мягкий совет пополнить проверки при малом остатке бонусных проверок.
* В `/stats` добавлена статистика продаж:
  * получено Stars;
  * количество покупок;
  * средний чек.

### Изменено

* Формулировка покупки заменена на более мягкую: «Пополнить проверки».
* Экран окончания проверок стал вести пользователя к пополнению.
* Сообщение после успешной оплаты стало более дружелюбным.

### Статус

* Бот готов к коммерческому MVP.
* Telegram Stars работают.
* Проверки после оплаты начисляются корректно.

### Платежная система

- Telegram Stars
- Robokassa (архитектура)
- Payment Gateway
- Payment Router

### Пользовательский профиль

- Статусы
- Автоматический месячный сброс
- Следующее обновление проверок

### UX

- Экран выбора способа оплаты
- Улучшен профиль
- Улучшены сообщения после оплаты

## v0.3.0 — SQLite и стабилизация MVP

### Добавлено

* Добавлен `database.py` для работы с SQLite.
* Добавлен файл базы данных `bot.db`.
* Добавлена таблица `user_checks`.
* Добавлены функции чтения и увеличения счетчика проверок.
* Добавлен `storage.py` как слой хранения данных.

### Изменено

* Счетчик бесплатных проверок переведен с памяти Python на SQLite.
* `user_profile.py` теперь получает данные через `storage.py`.
* `bot.py` не зависит от SQLite напрямую.

### Исправлено

* Исправлено двойное списание проверки в сценарии пограничной даты без времени.
* Исправлена проблема устаревших inline-кнопок выбора города.
* Улучшена обработка ошибок при выборе города.
* Убраны дублирующиеся тексты и лишний неиспользуемый код.

### Статус

* Бот запускается.
* Лимиты сохраняются после перезапуска.
* SQLite подключен и протестирован.
* MVP стабилен.
