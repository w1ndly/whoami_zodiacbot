# CHANGELOG

## v0.4.1 — Декомпозиция bot.py

### Добавлено

* Создан `handlers/callbacks.py`.
* Создан `handlers/birth.py`.

### Изменено

* Все команды вынесены из `bot.py` в отдельные обработчики.
* Вынесена обработка inline-кнопок.
* Вынесена обработка состояния `waiting_for_time`.
* Вынесена обработка состояния `waiting_for_place`.
* Вынесена обработка состояния `waiting_for_transition_place`.
* `bot.py` стал значительно компактнее и отвечает преимущественно за запуск приложения и координацию модулей.

### Исправлено

* Исправлен циклический импорт при переносе callback-обработчиков.
* Исправлена повторная регистрация Router в Aiogram.

### Статус

* Архитектура проекта стала модульной.
* Логика работы бота не изменилась.
* Все сценарии протестированы и работают.


## v0.3.1 — Постоянное хранение данных

### Добавлено

* Автоматическая инициализация SQLite при запуске бота.
* Поддержка Railway Volume для хранения базы данных.
* Автоматический выбор пути к базе через переменные окружения.

### Изменено

* База данных больше не зависит от временной файловой системы контейнера.
* Лимиты пользователей сохраняются после Redeploy.
* `database.py` переведен на безопасную работу через `with sqlite3.connect(...)`.

### Исправлено

* Исправлена ошибка `OperationalError: no such table: user_checks`.
* Исправлен сброс счетчиков после Redeploy на Railway.

### Статус

* SQLite полностью внедрен.
* Постоянное хранение данных работает.
* Бот готов к дальнейшему развитию.


## v0.3.0 — SQLite и стабилизация MVP

### Добавлено

* Добавлен `database.py` для работы с SQLite.
* Добавлен файл базы данных `bot.db`.
* Добавлена таблица `user_checks`.
* Добавлены функции чтения и увеличения счетчика проверок.
* Добавлен `storage.py` как слой хранения данных.

### Изменено

* Счетчик бесплатных проверок переведен с памяти Python на SQLite.
* `user_profile.py` теперь получает данные через `storage.py`.
* `bot.py` не зависит от SQLite напрямую.

### Исправлено

* Исправлено двойное списание проверки в сценарии пограничной даты без времени.
* Исправлена проблема устаревших inline-кнопок выбора города.
* Улучшена обработка ошибок при выборе города.
* Убраны дублирующиеся тексты и лишний неиспользуемый код.

### Статус

* Бот запускается.
* Лимиты сохраняются после перезапуска.
* SQLite подключен и протестирован.
* MVP стабилен.
