## Правила разработки

Работаем только маленькими шагами.

Каждый этап:

1. Проектирование.
2. Один небольшой кодовый блок.
3. Отдельное тестирование.
4. Подключение к проекту.
5. Проверка запуска.
6. Если этап крупный:

   * Restart Railway;
   * Redeploy Railway.
7. Обновление CHANGELOG.
8. Обновление PROJECT_STATE.

Никогда не исправлять две проблемы одновременно.

Не делать большие рефакторинги.

Не переписывать bot.py без необходимости.

Главная цель — стабильный проект.
